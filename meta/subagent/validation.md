# Subagent 冷启动验证方式（产品视角）

目标：还原 opencode agent 从使用者视角带来的反馈，并将反馈转成产品优化决策。

## 验证流程

1. 准备：定义本轮验证目标与样本角色。
2. 执行：按固定任务脚本观察 agent 与用户交互链路。
3. 记录：结构化记录行为轨迹与用户原话。
4. 复盘：筛选高价值反馈并形成下一轮产品动作。

## 本轮聚焦

- 聚焦：边界理解、安装可达、首次价值路径、失败恢复。
- 排除：弱网专项、探索额度治理。

## 数据采集

### 定量

- 任务完成率
- 首次价值时间（TTFV）
- 失败次数与重试次数
- 人工介入次数

### 定性

- 用户原话（误解点、犹豫点、信任变化）
- 失败归因（用户如何理解失败原因）
- 下一步动作是否可自主决策

## 反馈判定

- 高价值反馈：高频 + 高影响 + 可复现 + 可验证。
- 不收敛反馈：低复现或方向冲突且无法按用户分层解释。

## 非技术用户冷启动鲁棒性（占位）

当前仅保留占位要求，用于团队内部对齐，不作为 release gate。

- 性质：`informational`（记录观察，不阻断发布）
- 观察点 1：是否能用一句话解释 cowork 的边界（协作接口层，不替代业务仓库规则）
- 观察点 2：是否能在最小引导下拿到首个有效结果（如 `clone init -> clone metadata`）
- 观察点 3：首次失败后是否能根据提示完成自救（给出下一步可执行动作）

## 收敛执行集（当前）

当前按最小正交集合执行 4 条验证：

1. 安装后可执行：`install.sh` 后 `cowork --version`。
2. 首次价值路径：`clone init -> clone metadata`。
3. 高频失败恢复：缺失 `COWORK_CLONE_REPO_URL` 时失败可解释。
4. 远端失败恢复：`clone version` 远端不可达时失败可解释。
